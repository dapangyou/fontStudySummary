   问题：用户在浏览器输入地址 出来界面 中间经过了哪些过程
   1.http请求节点
   2.Http响应阶段
   3.浏览器的渲染阶段
        浏览器渲染原理:
```js
        客户端发送请求  服务端接收请求  服务端给客户端返回一串字符串(字符串就是浏览器需要渲染的代码)
        浏览器如何解析代码?
            1.首先浏览器会开辟栈内存给代码执行提供环境
            2.同时会分配一个主线程来执行代码  (浏览器是多线程程序 但是只会分配一个线程来执行代码)
            在栈中执行代码的过程是: 从上到下 代码依次进栈 执行完成后的代码会出栈
            主程序代码是由一个线程执行 然后浏览器会重新分配一个线程去加载css js 音视频资源文件等
            这些css js 等资源文件会存放在一个任务队列(taskQueue)中。这些存放在任务队列中的任务会等待
            浏览器分配的线程来执行。(等待中的任务又分为微任务和宏任务)
              ###  线程会执行任务队列中的任务 执行完成后会再去查看任务队列中的任务再执行 如此往复
                   这个过程就是传说中的EventLoop(事件轮询)
            3.主线程自上而下执行完成之后,会形成一个DOM Tree (DOM树)
              CSS执行完成后会形成CssOM
              DOM Tree + CSSDOM = render Tree  渲染树
            4.LayOut回流: 计算所有窗口的位置
            5.重绘: 计算呈现的样式
            6.显示
            

```

```js
        性能优化:
            1.减少http请求的大小和次数
            2.所有的css文件最好只用一个 
                css文件如果有多个,任务队列的事件也多一些,线程需要处理更多的东西 浪费性能
            3.图片使用懒加载的方式进行加载
            4.音视频使用流文件的形式
```

```js
    回流(Reflow):  元素的大小或者位置发生了改变 触发重新布局 导致渲染树重新计算布局和渲染
    重绘(Repaint): 元素样式的改变(但是宽高、位置、大小不变)
    注意：回流一定会触发重绘，而回流不一定会重绘
    性能优化也要避免重绘和回流
    如何减少DOM重绘和回流
        1.不操作DOM
            vue react 基于数据和视图 不操作DOM 优化性能
            但是底层不操作DOM么?
                底层是通过 virtual DOM /DOM diff  最大的限度减少操作DOM
        2.分离读写操作
                box.style.width = '300px';
                box.style.height = '300px';
            现代浏览器不会一次性就马上的渲染  会等待后面的执行结果 如果后面的代码依然是回流操作
            会一并执行  只有一次回流操作
            但是如果是
             box.style.width = '300px';
             console.log('test......');
             box.style.height = '300px';
            这就是两次回流操作了 中间的console.log()  会打断第一次的回流  
            所以要读写分离
            读写分离的具体表现可以是
            避免使用style 进行批量处理
        3.集中批量处理样式
            集中批量处理的代码形式是
                box.style.cssText = "width: 200px;height: 300px"; //批量处理
                box.className = 'aa';
                // 缓存处理
                box.style.width = box.clientWidth + 10 + 'px';
                box.style.height = box.clientHeight + 10 + 'px'; //2次回流
                // box.clientWidth  box.clientHeight  每次获取的时候都会是一次回流 可以设置变量存起来再操作
```
