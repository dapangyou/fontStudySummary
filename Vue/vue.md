指令

```js
    v-once指令:有v-once指令只加载data中的数据一次 之后不再加载
               即使data中的数据改变了 界面上也不会有改变
    v-html指令:解析标签
    v-text指令: 在页面展示data中的数据
                这个指令和插值表达式的功能一样
    v-text指令和插值表达式的区别?
                v-text指令没有插值表达式好用 如果后面追加一个值
                那么v-text中会覆盖其他的值 插值表达式就会追加值
    v-pre指令:  加上该指令 就不会解析插值表达式了  插值表达式会原封不动的显示到页面上
    v-cloak指令: vue还没有被解析的时候,会直接显示插值表达式的内容,然后等到vue解析之后才会显示数据内容
                 加上v-cloak 然后style添加属性样式display: none 可不显示 看起来更加直观
```

url
```js
    url中的hash和history
    url中是栈结构 有两种形式 hash和history
    hash: location.hash
    history: 
        history.back() --->  history.go(-1)   ---> 退回一步
        history.forward()->  history.go(1)    ---> 前进一步
        history.pushState({},'','aa');        ---> 入栈
        history.popState({},'','bb');         ---> 出栈
        history.replaceState({},'','ccc');    ---> 将当前的url替换
```

路由懒加载
```js
    首先我们知道路由通常会定义很多不同的页面
    这些页面最后被打包在哪里了呢?一般情况下是放在一个js文件中的
    但是那么多页面放在一个js文件中 必然会造成这个js非常的大
    如果我们一次性从服务器请求下来这个页面 可能需要花费一定的时间 甚至用户的电脑上会出现短暂的白屏现象
    如何避免这种情况呢?
        路由的懒加载做了什么?
            路由懒加载的主要作用就是将路由对应的组件打包成一个个的js代码块
            只有在这个路由被访问到的时候才加载对象的组件

```
